# PPT Layout Reference Matcher

GPTs로 만든 페이지네이션을 입력하고, 각 페이지에 어울리는 PPT 장표 레이아웃/구조 레퍼런스를 빠르게 추천받기 위한 팀용 제작 보조 MVP입니다.

## 실행

```bash
npm install
npm run dev
```

Supabase 환경변수가 없으면 브라우저 로컬 저장소로 동작합니다. OpenAI 분석은 서버의 `OPENAI_API_KEY`를 우선 사용하며, 없을 때만 화면에서 임시 API 키를 입력해 테스트할 수 있습니다.

## MVP 범위

- GPTs 페이지네이션 붙여넣기, 미리보기, 교체/추가 반영
- 페이지네이션 섹션/페이지 CRUD
- 페이지별 추천 장표 필터링
- 페이지에 장표 레퍼런스 적용/해제
- 장표 레퍼런스 이미지 등록
- 정보구조/역할/레이아웃/스타일/구성요소 태그 정규화
- 디자인 레퍼런스 등록
- 선택 페이지와 선택 장표 기반 GPT 프롬프트 생성/복사
- 태그 겹침 기반 추천 장표 썸네일 탐색

## 운영 제안

- `supabase/schema.sql`은 로컬/내부 MVP용입니다. 팀 배포는 `supabase/team_schema.sql` 방향처럼 Supabase Auth, `team_id`, row-level mutation 기반으로 전환해야 합니다.
- 팀 배포에서는 OpenAI 키를 브라우저에 저장하지 말고 서버 환경변수 `OPENAI_API_KEY`로 관리하세요.
- PPT 원본 전체보다 장표 캡처 이미지를 등록하는 방식이 빠르고 안전합니다.
- 태그는 `병렬`, `비교`, `단계`, `카드형`, `타임라인`처럼 짧은 표준 단어로 유지해야 탐색 속도가 좋아집니다.
